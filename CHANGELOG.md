# 更新日志

### v2.35 (2026-05-03)

**新增**：
- 统一可配置化：`synonym_map`、`search_cache`、`cache_ttl_minutes`、`dead_entries_threshold` 四个新配置项，初始化阶段均支持自定义
- 同义词缓存：索引构建时 LLM 提取的同义词写入本地缓存，搜索时优先查表，生僻词再调 LLM
- 搜索缓存：搜索前查缓存（问题 md5 → 结果），TTL 内命中直接返回，减少重复搜索的 LLM 调用

**优化**：
- api_helper.py 加 CLI 入口：python3 api_helper.py <config.json> <method> <path> [body]，支持 get/post/put/delete/parallel_get/get_data/fetch_all_docs
- 分段策略从「2000 字一刀切」改为「按 Markdown 标题切分 → 按段落切分 → 按字数兜底」，每段带章节标题

**修复**：
- 429 整点等待改为立即暂停通知用户：`api_helper.py` 触及 5000/h 不再 `time.sleep`，直接抛异常 → Agent 保存进度通知用户整点后重试
  - 全链路 6 处同步更新：`SKILL.md` 错误处理表、`api_reference.md` 速率控制/429表、`search_flow.md` 搜索步骤/清理流程、`index_flow.md` 批量构建
- 子库选择策略重写：新增 `active_index_book` 配置项指向当前写入子库，不再硬编码 `index_books[0]`，容量达标自动创建新子库并更新指针
- 配置初始化流程重设计：Token 验证通过后自动检测并创建缺失的 `index_master_book`、`index_books`、`default_book` 知识库，无需用户手动在语雀创建
- `config.example.json` 补充字段注释，标注「首次配置时会自动创建」
- README 快速开始说明更新：一站式配置，只需提供 Token
